Un petit mail de synthèse pour préciser les attendus du premier livrable du projet fullstack : Le document de conception

Ce document doit structurer votre réflexion technique et fonctionnelle selon les axes suivants :

1. Vision Fonctionnelle
Motivation : Description "business" de l'application (objectifs, besoins adressés, valeur ajoutée).
Profils utilisateurs : Identification des différents types d'utilisateurs.
Backlog (UC/User Stories) : Liste numérotée et priorisée. Un focus particulier est attendu sur les User Stories du MVP (version 1.0.0).
2. Organisation et Planning
Planning de Sprints : Incluant un "Sprint 0" (mise en place de l'environnement DevOps : dépôts, CI/CD, gestion des tickets).
3. Architecture et Stack Technique
Architecture logicielle : Schéma et description des composants (Front-end, Back-end, Kafka, Base de données, API externes, etc.).
Plateforme DevOps : Détail de la chaîne de production (Gestion du code, stratégie de packaging, pipeline de Continuous Delivery, gestion des Merge Requests et dépôt d'artefacts).

Modalités de rendu
Le document est à fournir dans l'arborescence du dépôt projet.

Le 26 février, je prendrai la version qui sera présente sur le dépôt du projet et la mettrai de côté. Ce livrable servira à la première évaluation de notre projet.

Si vous avez des questions n'hésitez pas à me les faire parvenir via mail.

Bon courage à tous
